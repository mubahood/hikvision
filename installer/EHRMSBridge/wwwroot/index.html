    User Guide

    Quick reference for using the EHRMS Device Bridge

    What is this?

    The EHRMS Device Bridge connects Hikvision DS-K1T680 face recognition terminals to the EHRMS backend. It polls the device for access control events (face scans, card swipes), stores them in MySQL, and syncs them to your server via webhook.

    Getting Started

    • Use the Setup Wizard to install and configure everything step by step.
    • Go to Settings to update your device IP, database credentials, and webhook URL.
    • Use Run Services to start/stop the bridge and the Streamlit dashboard.

    Architecture

    The system has three components:

    • Bridge (hikvision_bridge.py) — Polls the Hikvision device every 2 seconds via ISAPI, saves events to MySQL + JSON, and POSTs them to your webhook.
    • Dashboard (dashboard.py) — A Streamlit web UI on port 8502 for monitoring events, syncing data, and managing the bridge.
    • Database — MySQL stores all events with sync status tracking and retry support.

    Configuration Reference

    • DEVICE_IP — IP address of the Hikvision terminal
    • DEVICE_USER / DEVICE_PASS — Device admin credentials (Digest Auth)
    • DB_HOST, DB_PORT, DB_USER, DB_PASS, DB_NAME — MySQL connection
    • WEBHOOK_URL — Your EHRMS backend endpoint (events POSTed here)
    • WEBHOOK_API_KEY — API key sent as X-API-Key header
    • POLL_INTERVAL — Seconds between device polls (default: 2)

    Troubleshooting

    • Bridge won't start: Check the Diagnostics page. Verify .env exists and Python + dependencies are installed.
    • No events appearing: Test device connection in Diagnostics. Ensure the device IP is correct and reachable.
    • Database errors: Verify MySQL is running and credentials are correct. Run mysql -u root -p < create_tables.sql.
    • Webhook sync failing: Check WEBHOOK_URL and API key. Look at bridge logs for HTTP error codes.
    • Permission denied: Run chmod +x start.sh stop.sh status.sh.

    Management Scripts

    • ./start.sh — Start bridge + dashboard
    • ./stop.sh — Stop all services
    • ./status.sh — Check running status
